# Contributing to Helix Code

Thanks for improving Helix Code. Bug fixes, UI polish, provider support, documentation, and testing feedback are all welcome.

## Before opening a pull request

1. Create a focused branch and keep unrelated changes out of it.
2. Run the checks locally:

   ```powershell
   npm.cmd run typecheck
   npm.cmd run build
   ```

3. Describe what changed, how you tested it, and include screenshots for visual changes when useful.
4. Never commit API keys, chat history, `AppData` contents, generated installers, or other personal data.
